# Luminary Docs

This folder is the lightweight knowledge base for the project.

The goal is to keep the important product and contributor context close to the code, so future work is not trapped in chat history or one person's memory.

## Contents

- [Architecture Notes](architecture.md)
  How the extension is put together and where state lives.
- [Release Checklist](release-checklist.md)
  The remaining steps to take Luminary from a cleaned local repo to a public release.

## Documentation Priorities

As Luminary grows, this folder should stay focused on durable project knowledge:

- architecture and storage rules
- import and export behavior
- release and packaging notes
- contributor-facing decisions that are hard to infer from code alone
